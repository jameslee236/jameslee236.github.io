Ruby Hashes vs. JavaScript Objects

09.13.15

This week at DBC, we made the switch into JavaScript. This has been exciting for me as it's helped me solidify concepts I've learned in Ruby and has given me an opportunity to start from the beginning. However, this has also been challenging in some regards because the terminology is different. One instance in which this is the case is between JavaScript objects and Ruby hashes. They are both similar in terms of functionality - they both hold data and organize it so that it can be used throughout your code. They are unique in that they can hold key/value pairs. For example, if you wanted to store someone's name, age, school, and major, you'd use an object or hash.

This is how you do this in JavaScript and how you access it: ```JavaScript var Object = { name: "James" age: 23 school: "UC Berkeley" major: "Economics" } ``` output=> Object["age"] = 23 This is how you would do the same thing in Ruby: ```Ruby hash = { name: "James", age: 23, school: "UC Berkeley", major: "Economics"} ``` output => hash[:age] = 23

The differences are noticeable through the two examples but they are essentially doing the same thing. You define a variable in both. You use 'var' in JavaScript but not in Ruby. The way you access the values are different as well but not by much. The real difference is in the built-in methods. Ruby is significantly easier to use because it has built-in sorting methods and other ways in which to manipulate the data within the hash. JavaScript on the other hand has fewer methods and is therefore harder for a beginner to understand.
